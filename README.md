# 极简录屏（Chrome 扩展）

一键录制 **整个屏幕 / 应用窗口 / 浏览器标签页**，可同时录系统声音与麦克风，叠加**摄像头画中画**、**框选区域**录制，导出 `webm` 或 `mp4`，录完即时回放并下载。纯前端、零依赖、不联网、不入库。

> 与同仓库的 `extension/`（网页实时双语字幕 + 学习笔记）是两个独立工具：那个只抓某个标签页的音频用于做字幕，这个是通用录屏。

## 加载

1. Chrome 打开 `chrome://extensions` → 右上「开发者模式」打开。
2. 「加载已解压的扩展程序」→ 选本 `screen-recorder/` 目录。
3. 工具栏出现 🎬 图标。

## 使用

1. 点扩展图标 → 「打开录制台」（会开成一个独立标签页，**关闭弹窗不影响录制**）。
2. 按需勾选：录麦克风 / 录系统声音 / **摄像头画中画**（选角落位置）/ **区域录制**；选导出格式（webm / mp4）、帧率、码率。
3. 点 **开始录制** → 在系统弹出的选择器里挑屏幕 / 窗口 / 标签页。
   - 想要里面的声音：在选择器里记得勾 **「分享音频 / Share audio」**（整屏分享系统声、标签页分享该页声）。
4. **区域录制**：选源后预览里会让你 **按住拖拽框选** 一块区域，点「确认并开始」只录这块（或「录整个画面」跳过）。
5. 录制中可 **暂停 / 继续 / 停止**；也可以点 Chrome 自带的「停止共享」条，会自动收尾。
6. 停止后：上方播放器变成可回放的录像，点 **⬇ 下载录像** 保存 `录屏-时间.webm`（或 `.mp4`）。

## 设计要点

- 录制跑在独立扩展页面 `recorder.html` 里，而不是 popup —— 因为 popup 关闭会立刻中断 `getDisplayMedia`。
- 用 `navigator.mediaDevices.getDisplayMedia()` 调系统原生选择器，覆盖整屏/窗口/标签页三种来源。
- **摄像头画中画 / 区域录制**：把屏幕画面（按需裁剪）+ 摄像头重绘到 `<canvas>`，再 `canvas.captureStream()` 录制；纯全屏不开这俩时走直通流，画质最高。
- **后台不卡**：录屏时录制台标签页在后台，`rAF`/`setInterval` 会被节流到 ~1fps，所以用 **Web Worker 定时器** 驱动每帧绘制（Worker 不受可见性节流）。
- 系统音 + 麦克风用 `AudioContext` 混成一条音轨；只有一个来源时直接用那条。
- **mp4**：新版 Chrome 的 `MediaRecorder` 原生支持录 mp4（H.264）；做特性检测，不支持时自动回退 webm 并提示。webm 走 vp9/opus 优先（回退 vp8）。每秒切片，长录不丢数据。

## 文件

| 文件 | 作用 |
|---|---|
| `manifest.json` | MV3 清单，仅申请 `tabs` 权限 |
| `popup.html / popup.js` | 工具栏弹窗，负责打开/聚焦录制台标签页 |
| `recorder.html / recorder.js` | 录制台：选源、录制、混音、计时、预览、下载 |
| `icons/` | 16/48/128 图标 |

## 说明 / 限制

- 受 DRM 保护的内容（部分流媒体）画面会黑屏，这是浏览器层面的保护，属正常。
- macOS 首次录整屏需在「系统设置 → 隐私与安全性 → 屏幕录制」里给 Chrome 授权；用摄像头/麦克风也会弹一次权限。
- mp4 选项在不支持的旧 Chrome 上会自动回退 webm（状态栏会提示），仍可用 ffmpeg 转：`ffmpeg -i 录屏.webm out.mp4`。
- 区域录制是「整源捕获后裁剪」，所以选的区域越小、`canvas` 越小，文件也越小。
