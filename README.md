# 极简录屏（Chrome 扩展）

一键录制 **整个屏幕 / 应用窗口 / 浏览器标签页**，可同时录系统声音与麦克风，录完即时回放并把 `webm` 下载到本地。纯前端、零依赖、不联网、不入库。

> 与同仓库的 `extension/`（网页实时双语字幕 + 学习笔记）是两个独立工具：那个只抓某个标签页的音频用于做字幕，这个是通用录屏。

## 加载

1. Chrome 打开 `chrome://extensions` → 右上「开发者模式」打开。
2. 「加载已解压的扩展程序」→ 选本 `screen-recorder/` 目录。
3. 工具栏出现 🎬 图标。

## 使用

1. 点扩展图标 → 「打开录制台」（会开成一个独立标签页，**关闭弹窗不影响录制**）。
2. 按需勾选「录麦克风 / 录系统声音」、选帧率与码率。
3. 点 **开始录制** → 在系统弹出的选择器里挑屏幕 / 窗口 / 标签页。
   - 想要里面的声音：在选择器里记得勾 **「分享音频 / Share audio」**（整屏分享系统声、标签页分享该页声）。
4. 录制中可 **暂停 / 继续 / 停止**；也可以点 Chrome 自带的「停止共享」条，会自动收尾。
5. 停止后：上方播放器变成可回放的录像，点 **⬇ 下载录像** 保存 `录屏-时间.webm`。

## 设计要点

- 录制跑在独立扩展页面 `recorder.html` 里，而不是 popup —— 因为 popup 关闭会立刻中断 `getDisplayMedia`。
- 用 `navigator.mediaDevices.getDisplayMedia()` 调系统原生选择器，覆盖整屏/窗口/标签页三种来源。
- 系统音 + 麦克风用 `AudioContext` 混成一条音轨；只有一个来源时直接用那条。
- `MediaRecorder` 按 vp9/opus 优先编码（回退 vp8），每秒切片，长时间录制不丢数据。

## 文件

| 文件 | 作用 |
|---|---|
| `manifest.json` | MV3 清单，仅申请 `tabs` 权限 |
| `popup.html / popup.js` | 工具栏弹窗，负责打开/聚焦录制台标签页 |
| `recorder.html / recorder.js` | 录制台：选源、录制、混音、计时、预览、下载 |
| `icons/` | 16/48/128 图标 |

## 说明 / 限制

- 受 DRM 保护的内容（部分流媒体）画面会黑屏，这是浏览器层面的保护，属正常。
- macOS 首次录整屏需在「系统设置 → 隐私与安全性 → 屏幕录制」里给 Chrome 授权。
- 输出固定 `webm`；要 `mp4` 可再用 ffmpeg 转：`ffmpeg -i 录屏.webm out.mp4`。
